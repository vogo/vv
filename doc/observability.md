# 可观测性

vv 的可观测性由四个独立子系统组成，它们共享一个事件总线，按需挂载。

| 子系统 | 用途 | 默认 |
|--------|------|------|
| Debug | 单次 LLM/工具 I/O 的逐次记录 | 关 |
| Trace | 结构化事件流 JSONL 落盘 | 关 |
| Budget | session/daily 维度的硬上限与告警 | 按需 |
| Hooks | 业务侧扩展（自定义计数、注入、转发） | 按需 |

四者互不依赖，可任意组合。所有组件都遵循"未启用 → 不构造 → 零开销"原则。

## 事件总线

vage 的事件总线是统一基础。代理在执行过程中发出标准事件（请求开始/结束、LLM 调用开始/结束、工具调用开始/结束、阶段进出等）。订阅者各自决定如何处理，互不感知。

vv 把以下三件事都注册到这条总线：

- 持久化 Session 的 `events.jsonl` 写入。
- Trace 的 JSONL 落盘。
- Session Tree 的 auto-enable 计数。

设计取舍是 **共用一条总线**：避免维护多套订阅模型。代价是所有订阅者必须容忍其他订阅者的延迟（异步 hook 缓解了这一点）。

## Debug

Debug 是开发期工具：当用户加上 `--debug` 时，每次 LLM 请求与每次工具调用的入参/回值都被 **完整记录**（不截断、不脱敏）。

不同模式下输出位置不同：

- 交互式 CLI → 写到磁盘文件（避免污染 TUI 渲染）。
- 单提示模式 → 走 stderr。
- HTTP / MCP → 走结构化日志。

Debug 装饰位于工具与 LLM 中间件链的 **最外层**：它看到的是组件实际收到的（已截断、已权限拦截后的）参数与（已包装的）结果。这种位置对调试"为什么模型看到了这个内容"特别有用。

## Trace

Trace 是生产观测：当 `trace.enabled` 启用时，事件流以结构化 JSONL 形式按"项目散列 + 会话 id"落盘，支持文件大小轮转。

设计要点：

- **异步落盘**：主请求路径不等待磁盘 I/O。
- **缓冲队列**：突发流量下短期缓存到内存，避免阻塞。
- **按项目分目录**：多项目场景下 trace 不会混在一起。
- **rotation**：单文件超限自动轮转，避免 unbounded 增长。

Trace 与 Session 子系统的差别：

| 维度 | Trace | Session |
|------|-------|---------|
| 用途 | 离线分析、复盘 | 用户可见的会话历史 |
| 形态 | 平铺 JSONL | 元数据 + 事件流 + 状态 |
| 生命周期 | 长期保留 + 轮转 | 跟随会话删除 |
| 用户可见 | 无 UI 入口 | CLI/HTTP 都能查 |

两者可以独立开关。

## Budget

预算是 vv 提供的 **硬性约束**，分两个维度：

- **session 级**：单次会话或单进程生命周期内的累积上限。
- **daily 级**：跨会话按日重置。

度量两个维度：

- token 数（对所有 provider 通用）。
- USD 成本（按模型价格表折算）。

设计要点：

- **硬阀门**：超过即拒绝下一次 LLM 调用，不管模型还想不想用。HTTP 模式下转化为 429 响应。
- **告警阈值**：达到上限的 80%（可配置）时发出告警事件，让 CLI 可以提示、HTTP 可以转发。
- **价格表可覆盖**：内置默认价格表覆盖主流模型；用户可在配置里加新条目或覆盖。
- **日预算用进程内计数**：重启清零。这是有意取舍——持久化日预算会带来跨进程协调，复杂度不值得。

## Hooks

vv 暴露了一组 hook 接口供业务侧扩展：

- 在每次代理 Run 前/后被调用。
- 可以做：自定义日志、metrics 上报、入参转换、错误重试逻辑。

vv 自己挂了一个最小的 logging hook，作为示例与默认。其他 hook 由用户自行注册。

## 三种成本视图

可观测性最常被问的问题是"花了多少钱"。vv 提供三种粒度：

| 视图 | 范围 | 实时性 |
|------|------|-------|
| 当前 turn 累计 token | 进行中的一轮 | 流式实时 |
| Cost tracker | 进程生命周期内总和 | 实时 |
| Budget tracker | 配额维度（session/daily） | 实时 + 告警 |

三种视图都基于同一份 LLM 中间件的统计数据，差别只在累加边界。

CLI 在屏幕底部实时显示前两种；HTTP 通过 `/v1/budget` 暴露第三种；trace 与 session 事件流让事后分析也能算出。

## 设计回顾

可观测性子系统的核心约束：

- **不污染主路径**：所有计数与落盘都在事件总线之外，主路径只发事件。
- **可独立开关**：四个子系统正交，按需启用。
- **统一事件类型**：所有订阅者看到同一组事件，复用 vage schema。

这种"事件优先 + 旁路订阅"的架构让新增观测维度变得简单——再加一个订阅者即可，不需要改主路径。
